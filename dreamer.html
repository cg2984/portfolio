Goals

1. Learn how to connect a backend with a front end

2. Design for mobile.

Project Brief

For this project I was tasked with making a social media site. The website was coded in react, and hosted on netlify. The API that I called the post data from was made with an express server, and google firebase. I authenticated users in firebase as well.

via GIPHY

Code

I coded the frontend in React, and the backend in Express and Firebase. I created an API with all of the post data . The post data comes from an HTML form in the REact app. That data is stored in firebase, and hosted on Heroku via an Express server. That data is then called using Axios in the React app and displayed on the website. I authenticated the users with Firebase as well.

Wireframe

My main inspiration was instagram. I wanted to make an app that was like instagram, but without the likes or the comments. The brand of this social media site would be something more contemplative, and les concerned with what people think.

Lo-Fi Prototype

I changed the small squares into larger posts since they were too small on mobile. I considered a gradient, but came to the conclusion that a solid background would be better since it was cleaner. In this stage I was mostly attempting to nail down the different assets that I was going to code.

Hi-Fi v1

When I actually began styling my React app I came into a few challenges. I was having trouble figuring out how to add pictures and a chat to my site. Since I had limited time, I decided to pivot a little bit. I changed the post format, since I decided not to add picture, and got rid of the chat feature.

HiFi v2

Since I was limited to text, I began thinking about journals. This website would essentially have to be a way to only share your thoughts and ideas. I was inspired by dream journals, which are records people keep of the dreams that they keep after they wake up from sleep. I decided to change my idea. I kept the same layout, but I changed out the colors, typeface, and the header. I changed the colors to periwinkle to evoke a dreamy feeling. I considered changing the color of the tposts as a well, but I felt that having them a pale yellow on a blue background differentiated the posts from the inputs. Lastly, I changed the font to Quicksand because it has rounded, soft edges and I wanted my new app to be soft and dreamy.
Codewise, I was able to implement a date marker to the posts. For my original idea it wasn't as important, but since users would be tracking the dreams they had I figured it would be more important. To implement this feature, I had a variable record the date. Then, whenever a post is sent out, I send that days date with the rest of the post data (like the title and the text) to firebase for it to be called when the post is displayed.

Final Version

In this final version I changed the header to a softer font. I wanted something rounded, but not juvenille. I also finalized the look of the posts, and the footer.
